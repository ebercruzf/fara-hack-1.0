Branch   : bugfix/sovereign-2.0-day2
Hashtag  : #AgentXHackathon

Summary
-------
Replaced the HiveMind 7-agent software-dev swarm (which dispatched
COMMANDER/BLUEPRINT/BUILDER/...) with a focused 4-agent SRE bug-triage
pipeline orchestrated from BugReportController via DirectAgentExecutor
(the same execution path used by `/agent <id>` in the CLI). The new
pipeline runs end-to-end, sends a real Gmail SMTP notification to the
on-call engineer, and creates a ticket in the local store — all in
~3 minutes per submission with qwen3.5:35b-a3b on local Ollama.

Pipeline shape (Java orchestration, Java 25 Virtual Threads)
------------------------------------------------------------

   STEP 0  : LlmTriageClient (vision adapter, qwen3.5:35b-a3b multimodal)
                                |
                                v
   STEP 2  : triage-coordinator           (sequential gate)
                                |
                                v
   STEP 2.5 ║ STEP 6  : forensic-analyst  ║  mitigation-engineer
                                |  (parallel via Thread.ofVirtual())
                                v
   STEP 3+4 : triage-broker               (sequential, email_send)
                                |
                                v
   STEP 5  : TriageStatusWatcher (existing actor, polls every 30s)

Each agent is loaded by AgentTemplateManager from its YAML in
workspace/.fararoni/config/agentes/, and invoked by DirectAgentExecutor
with the same toolset filter the CLI `/agent` command uses.

        triage-coordinator       (sequential gate)

        triage-broker            (sequential, dispatches email)

HACKATHON COVERAGE (5 mandatory + 2 differentiators)
────────────────────────────────────────────────────────────────────────

  Step 1  Submit via UI                  BugReportController        DONE
  Step 2  Triage + summary               triage-coordinator agent   DONE
  Step 2.5 Forensic auto-classification  forensic-analyst agent     DONE *
  Step 3  Create ticket                  TicketStore.create()       DONE
  Step 4  Notify team (email)            triage-broker + Gmail SMTP DONE
  Step 5  Notify reporter on resolved    TriageStatusWatcher actor  DONE
  Step 6  Sentinel Mitigation Proposal   mitigation-engineer agent  DONE *

  * Steps 2.5 and 6 are the differentiators. They run because the agent
    YAMLs are loaded and reasoning, but the supporting tools
    (forensic_memory_search, sentinel_audit, diff_generate, etc.) are
    declared in the YAMLs but NOT yet registered in the core
    ToolRegistry. The agents currently degrade gracefully (return
    null/N-A verdicts). Wiring those tools is core-side work that does
    NOT block the AgentX submission.

KNOWN GAPS / FOLLOW-UPS (post-hackathon)
────────────────────────────────────────────────────────────────────────

1. Tools declared in agent YAMLs but not in core ToolRegistry:
     code_search, arcadedb_query, forensic_memory_search, sentinel_audit,
     diff_generate, ticket_create, arcadedb_write, mcp_dispatch
   DirectAgentExecutor filters them silently. The 4-agent pipeline still
   runs end-to-end because the critical tools (fs_read, email_send)
   ARE registered. Wiring the missing tools turns the differentiator
   steps from "graceful degradation" into "full evidence with queryIds".

2. executionMode handling: ReactiveSwarmAgent.start() reads
   `template.metadata().get("executionMode")`, not the top-level field.
   Our 4 agents end up subscribed to swarm.task.* topics on boot.
   Harmless (nothing publishes there in our flow), but for cleanliness
   we should put `metadata: { executionMode: direct }` in the YAMLs.

3. Boot logs still show the 7 default-team agents (commander, blueprint,
   builder, operator, intel, strategist, sentinel) being loaded.
   They are unused by our pipeline but still occupy memory. Could be
   removed from the auto-provisioned default list to clean the boot
   trace for the demo video.

4. examples/ directory is still empty. To capture as evidence for
   AGENTS_USE.md sections 6 (observability) and 7 (security):
     examples/log-sample.txt          <- docker compose logs fara-hack
     examples/trace-sample.json       <- WS Live Feed export
     examples/tickets-snapshot.json   <- /api/triage/tickets
     examples/email-screenshot.png    <- the Gmail message
     tests/security/*.json            <- prompt injection fixtures

5. Demo video (3 min, English, #AgentXHackathon hashtag) — not yet
   recorded. PITCH.md has the script ready.
